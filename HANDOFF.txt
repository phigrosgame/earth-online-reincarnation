【一句話】
一款「AI 即時生成劇情」的網頁人生模擬遊戲。從一個普通人從 0 歲活過一生，
死亡後把累積的代幣化為詞條，越輪迴越強，目標是打破平凡（變首富、乃至超凡）。
純前端、單機、localStorage 存檔，劇情由 Google Gemini 生成。

  全程詳細 log 到 console（F12 可診斷 HTTP 狀態 / finishReason / JSON 解析）。

4. 核心玩法設計（目前狀態）
------------------------------------------------------------------
A. 世界（REALMS）
   mortal 現代凡人(預設) / rich 豪門 / noble 王朝 / cultivator 修仙 / immortal 神明 / chaos 混世界
   - 解鎖用「道果」(B幣)：rich24 / noble56 / cultivator120 / immortal300 / chaos40
   - chaos 混世界：解鎖任意 3 世界後開放；投生前「手動勾選」要混入哪些已解鎖世界 + 可選隨機詞條。
   - 每個世界有 shown(顯性顯示) 與 hidden(隱藏) 兩組數值，healthKey 指定「健康」對應鍵
     （凡人=健康、修仙=壽元、神明=法則）。

B. 顯性 vs 隱藏數值（重點機制）
   - 顯性（玩家看得到）：財富/名聲 等現實可見的。
   - 隱藏（平常看不到）：健康、運氣、智慧、心性…。只能靠事件「窺見」(maybeRevealHidden，每章40%機率，
     健康檢查/考試/算命 顯示當下文字分級)。
   - 給 AI 時：顯性給數字、隱藏給文字分級（levelWord / healthWord），讓 AI 理解但玩家看不到精確值。

C. 章節批次生成（省等待）
   - chapterPrompt() 一次生成：daily[]（接下來幾年的日常小事，可帶輕量 opts）+ major（重大事件，4選+1手輸）。
   - 日常小事自動逐條播放、不阻擋流程；有 opts 的小事內嵌可點但「不暫停」，不點＝預設沒選。
   - 大事件選完 → 整個事件＋你的選擇用 addFlowMajor() 永久寫進「人生」時間流。
   - 年齡從 0 歲開始（startAge:0）。

D. 骰子四級危機判定（重要：避免 AI 只寫成功）
   - 凡是 gamble(成敗未定，如創業/打官司) 或 risky(危及生命) 的選項：
     程式先 rollTier() 擲出【大成功/小成功/失敗/慘敗】(依主導隱藏數值 key + 運氣)，
     再用 narrateTiered() 叫 Gemini「照這個已定的結果」生成劇情，AI 不准自己改成功。
   - 一般(非 staked)選項才直接用 AI 預寫的 outcome。

E. 三種非自然死亡（死亡管道彼此獨立，函式在 game.js）
   - 純意外（車禍/地震，與行為無關）：tickYears() 內每年 ACCIDENT_PER_YEAR=0.0007，
     一生(~78年)累積≈5%。
   - 報復橫死（行為推算）：結怨選項(makesEnemy) → addThreat() 累積威脅值 → 每章 rollRetaliation()
     依威脅值×運氣判定是否被暗殺。
   - 疾病病逝：過勞扣健康 + 自然衰減，健康<35 起 rollIllnessDeath() 機率漸增；健康低時提示詞要求
     daily 寫病徵(咳嗽/白髮)。
   - 死因會反映在結算標題與結語(DEATH_TITLE / deathSummary)。

F. 難度 / 基調
   - lifeToneLine()：第一世且無強力詞條 → 強制「再普通不過的人」(普通家庭/正常升學)，禁止天降橫財。
   - 重大挫折低頻：gs.setbacks 計數，提示詞要求一輩子約 1~3 次、滿 3 次不再安排。
   - 家庭不睦是「機率事件」非預設，多數人家庭普通和睦。

5. 詞條抽卡經濟
------------------------------------------------------------------
- 兩種代幣（變數名仍是 aCoin/bCoin，顯示名已改）：
  - 魂魄 (save.aCoin)：常用，抽詞條、轉永久基礎天賦。
  - 道果 (save.bCoin)：稀有，擴充攜帶格、高階保底抽卡、解鎖世界。
- 詞條 TRAITS[]：5 階稀有度 N/R/SR/SSR/UR，抽卡機率 55/28/12/4/1%，十連保底 SR+。
- 合成：同詞條集滿 5 張升一級（最高 Lv.5），加成隨等級×(1+0.4*(lv-1))。
- 詞條效果：passive(被動加隱藏數值/起始錢) + trigger(字串，寫進提示詞讓 AI 生成專屬劇情，
  如「重生者」知未來、「金手指」綁系統)。
- 攜帶格上限 save.carryCap（預設3，道果擴充）。

6. 財富 / 數值規則（最近一次大改，已現實化）
------------------------------------------------------------------
- 數值分類（game.js 頂部）：
  MONEY_STATS = {財富,家產} 以「元」計、無上限。
  CURRENCY_STATS = {靈石,信徒,命運值,官品,境界,神格} 各自尺度、無上限。
  FAME_STATS = {名聲,聲望,功勳,名望,民心} 無上限但「邊際遞減」。
  健康類(healthKey) 有上限 HEALTH_CAP=120（避免不死）；其餘隱藏數值無上限。
- bumpStat() 是統一的加減函式，applyEffects() 已改用它（修掉了「所有數值被 clamp 到 120」的舊 bug）。
- addFame(current,gain)：名聲越高同樣 +gain 實際增益越小（gain*12/(12+current)）。
- 起始財富：新生兒≈0；「富貴命」等詞條才給出生財富（金錢世界 startMoney*2,000,000*scale）。
  豪門家產起始 30,000,000。
- 顯示：fmtMoney/moneyText 大額用「萬/億」（例 123萬、25億）。
- 現實基準（已寫進提示詞，元世界）：普通人辛苦一生總資產≈100萬；千萬=富人；億=頂級富豪(極罕見)。

7. 代幣給予公式（結算 endLife）
------------------------------------------------------------------
- 先算財富成就 ach = wealthAchievement()：元世界 (log10(財富)-4)*60，夾0~320
  → 1萬=0 / 10萬=60 / 100萬=120 / 1000萬=180 / 1億=240 / 10億=300。
  非元世界用 √(財富/moneyMax)*250。
- 魂魄 aCoin = 50 + 重大抉擇數*14 + ach + 壽命比例*50（上限4000）。常用、給得多。
- 道果 bCoin（稀有，普通人一生頂多1）：
  ach≥180 +1；ach≥240 +1；ach≥300 +1；善終(非早逝且活過85%壽命) +1；上限5。

8. 已知狀況 / 待辦 / 注意
------------------------------------------------------------------
- AI 配合度（章節 JSON 格式、骰子結果是否被遵守、門檻邏輯）只能真機填金鑰實測，純靜態測不到。
- 進行中存檔 save.activeLife 為舊版欄位時可續玩（有相容處理），但舊存檔起點可能非0歲，建議開新局。
- 仍可調整：機率微調、世界平衡、更多詞條、超凡(immortal/cultivator)後期內容、UI 美化。
- 使用者偏好：要「能玩的成品」而非架構文件；每專案獨立資料夾；開發在 Linux、實玩在 Windows(雙擊或 Pages)。
- 改動規模大時先語法檢查＋機率模擬再 push；push 後提醒使用者撤銷外洩憑證。
